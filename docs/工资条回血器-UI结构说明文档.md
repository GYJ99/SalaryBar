# 工资条回血器（macOS 顶部栏版）UI 结构说明文档

## 1. 文档信息

- 文档名称：工资条回血器（macOS 顶部栏版）UI 结构说明文档
- 文档版本：v1.0
- 编写日期：2026-03-27
- 关联文档：`docs/工资条回血器-功能需求文档.md`

## 2. 文档目标

本文档用于定义“工资条回血器”在 macOS 顶部栏应用中的界面结构、主要交互、视觉层级与组件拆分，作为后续 UI 设计、SwiftUI 视图实现和交互验收的依据。

本文档聚焦：

- 顶部栏展示方式
- 弹出面板布局
- 设置窗口信息架构
- 关键组件状态
- 视觉风格与尺寸建议

本文档不覆盖：

- 具体计算公式实现细节
- 数据持久化结构细节
- 打包、签名、DMG 产物流程

## 3. 产品界面总览

应用共包含 4 个核心界面层：

1. 顶部栏状态项
2. 顶部栏弹出详情面板
3. 设置窗口
4. 首次启动初始化引导

其中用户最高频接触的是前两层：

- 平时只看顶部栏状态项
- 需要细节时点击展开详情面板

## 4. 界面结构总图

### 4.1 信息架构

```text
工资条回血器
├── 顶部栏状态项
│   ├── 图标
│   ├── 实时金额
│   └── 状态提示
├── 弹出详情面板
│   ├── 顶部状态条
│   ├── 收入主卡片
│   ├── 当前目标进度
│   ├── 快捷操作区
│   └── 成就目标列表
├── 设置窗口
│   ├── 薪资设置
│   ├── 工作时段设置
│   ├── 显示设置
│   ├── 目标设置
│   └── 系统设置
└── 首次启动引导
    ├── 欢迎页
    ├── 薪资配置
    ├── 工作时间配置
    └── 完成页
```

### 4.2 交互入口关系

- 用户点击顶部栏状态项：打开详情面板
- 用户在详情面板点击“设置”：打开设置窗口
- 用户在详情面板点击“暂停/继续”：直接改变计时状态
- 用户首次打开应用：优先进入初始化引导
- 用户配置完成后：回到顶部栏常驻模式

## 5. 顶部栏状态项设计

### 5.1 目标

顶部栏状态项的目标不是承载复杂信息，而是以最小干扰成本提供“我现在赚了多少钱”的即时反馈。

### 5.2 展示内容

首版建议提供 3 种显示模式：

#### 模式 A：仅金额

示例：

- `￥12.53`
- `￥108.2`

适用于顶部栏空间紧张的场景。

#### 模式 B：图标 + 金额

示例：

- `¥ 12.53`
- `💰 88.20`

适用于希望保留产品辨识度的场景。

#### 模式 C：文案 + 金额

示例：

- `已赚 ￥12.53`
- `今日 ￥88.20`

适用于强调语义的场景，但宽度占用最大。

### 5.3 状态定义

顶部栏状态项应至少支持以下状态：

- 工作中
- 暂停中
- 非工作时段
- 未完成配置

### 5.4 各状态文案建议

#### 工作中

- 显示实时增长金额

#### 暂停中

- `暂停中`
- 或 `￥12.53 | 暂停`

#### 非工作时段

- `未开工`
- 或展示今日最终金额但不再变化

#### 未完成配置

- `去设置`
- 或 `未配置`

### 5.5 布局建议

- 图标位于左侧
- 金额为主文本，单行显示
- 文本垂直居中
- 避免金额变化导致控件宽度频繁跳动

### 5.6 交互规则

- 单击：展开或收起详情面板
- 右击或长按：可打开快捷菜单
- 鼠标悬停：可显示当前状态的简要说明

### 5.7 快捷菜单建议

- 打开详情
- 暂停 / 继续
- 打开设置
- 开机启动
- 退出应用

## 6. 详情面板设计

### 6.1 面板定位

详情面板是产品最重要的视觉载体，需承接参考图中的“治愈账单卡片”气质，同时适配 macOS 顶部栏弹出方式。

### 6.2 面板尺寸建议

首版建议默认尺寸：

- 宽度：420 至 500 pt
- 高度：560 至 680 pt

如果目标列表较长，列表区单独滚动，不让整面板无限增高。

### 6.3 面板结构

```text
详情面板
├── 顶部状态条
├── 今日收入主卡
├── 收入速率区
├── 当前目标进度区
├── 快捷操作按钮区
└── 成就目标列表区
```

## 7. 顶部状态条

### 7.1 作用

用于表达产品身份、当前状态与正在运行的时长信息。

### 7.2 内容结构

- 左侧：小徽标或产品标签
- 中部：产品标题，如“工资可视化”
- 右侧：当前状态标签

次级信息：

- 计时器，如 `00:18:32`

### 7.3 状态标签颜色建议

- 工作中：墨绿色底 + 浅色文字
- 暂停中：暖灰底 + 深色文字
- 非工作时段：浅棕底 + 深棕文字
- 未配置：低饱和橙色提示

## 8. 今日收入主卡

### 8.1 作用

作为整个详情面板的视觉中心，突出“今日已赚金额”。

### 8.2 布局结构

```text
今日收入主卡
├── 计时胶囊
├── 主金额
├── 副标题
├── 分隔线
├── 三列收入速率
├── 下一个目标提示
├── 目标进度条
└── 主操作按钮
```

### 8.3 元素说明

#### 计时胶囊

- 展示本次连续工作时长
- 胶囊式小组件，居中摆放

#### 主金额

- 今日已赚金额
- 字号最大，是视觉第一层级
- 金额字体建议具备复古账本感

#### 副标题

- 固定文案：`今日已赚`
- 或状态文案：`正在回血`

#### 三列收入速率

- 每秒
- 每分钟
- 每小时

每列包含：

- 标签
- 数值

#### 下一个目标提示

示例：

- `下一目标：一杯蜜雪冰城`
- `还差 ￥3.60`

#### 目标进度条

- 显示当前金额到目标金额的进度
- 进度条左侧可用深绿色点状装饰
- 进度完成后切换到下一个目标

#### 主操作按钮

- 工作中时显示：`收工休息`
- 暂停中时显示：`继续回血`

### 8.4 卡片视觉建议

- 大圆角白色卡片
- 浅阴影
- 深棕主字色
- 墨绿作强调色
- 上下留白充足

## 9. 成就目标列表区

### 9.1 作用

将抽象金额转成具象消费目标，增强“赚钱可感知性”。

### 9.2 结构

```text
成就区
├── 标题栏
│   ├── 标题
│   └── 解锁数量
└── 列表
    ├── 目标项 1
    ├── 目标项 2
    ├── 目标项 3
    └── ...
```

### 9.3 标题栏内容

- 标题：`搞钱成就`
- 副信息：`3 / 12 已解锁`

### 9.4 目标项结构

每个目标项包含：

- 左侧图标位
- 名称
- 金额
- 解锁状态

### 9.5 目标项状态

#### 未解锁

- 白底或浅底
- 低对比图标
- 名称和金额正常显示

#### 当前冲刺中

- 边框高亮
- 图标或金额使用强调色
- 可加轻微呼吸感动效

#### 已解锁

- 显示勾选、徽章或彩带感装饰
- 可降低透明度，表示已完成

### 9.6 列表交互

- 支持滚动
- 点击可查看详情或编辑入口
- 长按 / 右击可打开快捷操作

## 10. 快捷操作区

### 10.1 放置位置

建议置于主卡片底部，或主卡片下方单独一行。

### 10.2 建议按钮

- 暂停 / 继续
- 打开设置
- 重置今日计时

首版若需收敛复杂度，默认只保留：

- 主按钮：暂停 / 继续
- 次按钮：设置

## 11. 设置窗口设计

### 11.1 设计目标

设置窗口负责承载完整配置，不追求顶部栏的轻量感，重点是清晰、直接、易懂。

### 11.2 窗口结构建议

首版建议采用左侧导航 + 右侧内容区结构。

```text
设置窗口
├── 侧边栏
│   ├── 薪资设置
│   ├── 工作时段
│   ├── 显示设置
│   ├── 目标设置
│   └── 系统设置
└── 内容区
    └── 对应表单
```

### 11.3 薪资设置页

#### 内容项

- 薪资模式选择
- 薪资金额输入框
- 计薪周期选择
- 月薪换算方式说明
- 每秒 / 每分 / 每小时预览结果

#### 交互要求

- 用户输入后实时展示换算结果
- 公式说明不能隐藏太深

### 11.4 工作时段设置页

#### 内容项

- 上班开始时间
- 下班结束时间
- 午休开始时间
- 午休结束时间
- 每周工作天数
- 周末制度选择

#### 交互要求

- 时间段必须可视化校验
- 若开始时间晚于结束时间，应即时提示错误

### 11.5 显示设置页

#### 内容项

- 顶部栏显示模式
- 小数位数
- 是否显示图标
- 是否显示“今日已赚”文案
- 浅色 / 跟随系统

### 11.6 目标设置页

#### 内容项

- 目标列表
- 新增目标按钮
- 编辑目标
- 删除目标
- 拖拽排序
- 导入默认目标

### 11.7 系统设置页

#### 内容项

- 开机启动
- 通知开关
- 默认启动后是否自动开始计时
- 关闭窗口时是否仅隐藏

## 12. 首次启动引导

### 12.1 目标

尽快让用户完成最少配置，进入“能看到金额变化”的状态。

### 12.2 引导步骤建议

#### 第 1 步：欢迎页

- 简要说明产品作用
- 强调“常驻顶部栏，按秒回血”

#### 第 2 步：设置工资

- 选择月薪或时薪
- 输入金额
- 立即展示换算预览

#### 第 3 步：设置工作时段

- 设置上班/下班时间
- 设置午休

#### 第 4 步：完成页

- 告知“顶部栏已开始显示”
- 提供“立即查看”按钮

### 12.3 引导策略

- 支持跳过高级设置
- 首次流程尽量不超过 4 步
- 目标是在 1 分钟内完成配置

## 13. 关键用户流程

### 13.1 首次使用流程

```text
安装应用
→ 首次启动
→ 初始化引导
→ 完成工资和工作时间配置
→ 顶部栏开始显示金额
→ 点击查看详情面板
```

### 13.2 日常使用流程

```text
开机启动应用
→ 顶部栏显示当前状态
→ 工作中金额按秒增长
→ 点击展开详情
→ 查看进度与成就
→ 休息时手动暂停
→ 恢复时继续累计
```

### 13.3 调整配置流程

```text
顶部栏展开详情
→ 点击设置
→ 修改薪资或工作时段
→ 保存配置
→ 顶部栏和详情面板实时更新
```

## 15. 视觉规范建议

### 15.1 色彩方向

- 背景色：奶油米白
- 主文字：深棕色
- 强调色：墨绿色
- 分割线：暖灰色
- 禁用态：低饱和灰棕

### 15.2 字体方向

- 金额主数字：高识别度衬线或半衬线风格
- 正文信息：系统中文字体或可读性强的无衬线
- 顶部栏文字：优先使用系统字体，保证清晰

### 15.3 圆角与阴影

- 主卡片圆角建议 20 至 28 pt
- 小胶囊圆角建议 999 pt
- 阴影轻量，不做重拟物

### 15.4 动效建议

- 金额跳动更新只做轻微淡入/数值过渡
- 解锁目标时做一次性提示动效
- 面板展开/收起应柔和，不拖沓

## 16. 适配与边界场景

### 16.1 深色模式

首版可以默认优先适配浅色主题，但在深色模式下必须保持可读性，不允许出现文字对比不足。

### 16.2 顶部栏拥挤场景

- 当顶部栏空间不足时，自动退化为仅金额模式
- 当金额过长时，优先缩短小数位而不是截断货币符号

### 16.3 超长目标列表

- 列表区独立滚动
- 标题栏固定

### 16.4 未配置状态

- 详情面板显示空态引导
- 主按钮文案改为：`去完成设置`

## 17. UI 验收要点

- 顶部栏状态项在工作中、暂停中、未配置三种状态下都清晰可辨
- 详情面板信息层次明确，主金额是视觉中心
- 主卡片与成就列表具备参考图中的温和复古质感
- 设置窗口表单结构清楚，用户能快速找到工资与工作时段配置
- 首次使用能在较短时间内完成设置并看到实时金额
